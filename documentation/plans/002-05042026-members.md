# Members Entity

## Context

The app needs members — the people who participate in teams (players, coaches, etc.). A member can belong to multiple teams with a specific role in each. Members are optionally linked to a user account. The club relationship is derived through team assignments.

## Data model

### `member_statuses` lookup table

| Column | Type   | Constraints      |
| ------ | ------ | ---------------- |
| id     | uuid   | PK, auto         |
| label  | string | required, unique |

### `members` table

| Column     | Type                | Constraints                    |
| ---------- | ------------------- | ------------------------------ |
| id         | uuid                | PK, auto                       |
| user_id    | uuid                | FK → users, nullable           |
| status_id  | uuid                | FK → member_statuses, nullable |
| first_name | string              | required                       |
| last_name  | string              | required                       |
| birth_date | date                | nullable                       |
| license    | string              | nullable                       |
| gender     | enum (male, female) | required                       |
| year       | integer             | required (registration year)   |
| created_at | timestamp           | auto                           |
| updated_at | timestamp           | auto                           |

### `team_assignments` join table

| Column     | Type                                      | Constraints            |
| ---------- | ----------------------------------------- | ---------------------- |
| id         | uuid                                      | PK, auto               |
| team_id    | uuid                                      | FK → teams, required   |
| member_id  | uuid                                      | FK → members, required |
| role       | enum (player, coach, assistant, sparring) | required               |
| created_at | timestamp                                 | auto                   |

Unique constraint on (team_id, member_id) — a member can only have one role per team.

## Permissions

| Action                  | admin | manager | user     |
| ----------------------- | ----- | ------- | -------- |
| List/get members        | ✅    | ✅      | own only |
| Create member           | ✅    | ✅      | ❌       |
| Edit member             | ✅    | ✅      | own only |
| Delete member           | ✅    | ❌      | ❌       |
| Manage team assignments | ✅    | ✅      | ❌       |
| CRUD member_statuses    | ✅    | ❌      | ❌       |

"own only" = user can only access/edit the member linked to their user_id.

### 2. Member statuses CRUD routes (`/member-statuses`)

- `GET /` — list all (admin, manager)
- `POST /` — create (admin)
- `PUT /:id` — update (admin)
- `DELETE /:id` — delete (admin)

### 3. Members CRUD routes (`/members`)

- `GET /members` — list all, filter by `?teamId=` (admin, manager; user sees own only)
- `GET /members/:id` — get by ID (admin, manager, or own member)
- `POST /members` — create (admin, manager)
- `PUT /members/:id` — update (admin, manager, or own member)
- `DELETE /members/:id` — delete (admin only)

For "own member" checks: middleware compares `req.user.id` against `member.user_id` from the DB.

### 4. Team membership routes

- `GET /teams/:teamId/members` — list members of a team (admin, manager)
- `POST /teams/:teamId/members` — add member to team with role (admin, manager)
- `DELETE /teams/:teamId/members/:memberId` — remove member from team (admin, manager)
