# Admin Pages

## Context

The API has full CRUD for clubs, teams, and members but the PWA has no admin interface. We need admin pages to manage these resources, accessible only to users with the `admin` role.

## Routes

- `/admin` — admin dashboard (links to sub-pages)
- `/admin/clubs` — list clubs
- `/admin/clubs/new` — create club
- `/admin/clubs/:id` — edit club
- `/admin/teams` — list teams (filter by club)
- `/admin/teams/new` — create team
- `/admin/teams/:id` — edit team
- `/admin/members` — list members (filter by team)
- `/admin/members/new` — create member
- `/admin/members/:id` — edit member

## Plan

### 1. Admin route guard

- **New:** `apps/pwa/src/layout/AdminGuard.tsx`
  - Wraps admin routes, checks `user.role === 'admin'`
  - Redirects to `/` if not admin
  - Uses `useAuth()` hook

### 2. Admin layout

- **New:** `apps/pwa/src/layout/AdminLayout.tsx`
  - Sidebar with admin navigation (Clubs, Teams, Members)
  - Uses Mantine `AppShell` with nested `<Outlet />`
  - Content area max-width: 1200px, right aligned
  - Back to app link

### 3. Admin nav link

- **Modify:** `apps/pwa/src/layout/AppNavigation.tsx`

### 4. API hooks

  - Add admin routes wrapped in `AdminGuard` and `AdminLayout`

## Patterns

- **List pages:** Mantine `Table` with `useQuery` to fetch data, `ActionIcon` for edit/delete
- **Form pages:** `useForm` from `@mantine/form`, `useMutation` for save, `useParams` for edit mode
- **Delete:** confirmation modal via `modals.openConfirmModal` from `@mantine/modals`
- **Notifications:** `notifications.show()` for success/error feedback
- **API calls:** Bearer token from `useAuth()`, `fetch` with JSON body
