# Add User Roles

## Context
Users currently have no role — all routes are unprotected. We need global roles (`admin`, `manager`, `user`) with authorization middleware to control access to club and team CRUD operations.

## Role Permissions

| Action | admin | manager | user |
|--------|-------|---------|------|
| List users | ✅ | ✅ | ❌ |
| Get user | ✅ | ✅ | ❌ |
| Create user | open (registration) | open | open |
| Edit user | ✅ (any) | self only | self only |
| Delete user | ✅ (any) | self only | self only |
| Create club | ✅ | ❌ | ❌ |
| Edit club | ✅ | ✅ | ❌ |
| Delete club | ✅ | ❌ | ❌ |
| Create team | ✅ | ✅ | ❌ |
| Edit team | ✅ | ✅ | ❌ |
| Delete team | ✅ | ❌ | ❌ |
| List/get clubs & teams | ✅ | ✅ | ❌ |

- First user ever created gets `admin` role
- All subsequent users get `user` role by default

## Plan

### 1. Migration: add `role` to users
- **New:** `apps/api/src/migrations/20260405_add_role_to_users.ts`
- Add `role` enum column (`admin`, `manager`, `user`), default `user`
- Backfill existing users with `user`

### 2. JWT auth middleware
- **New:** `apps/api/src/middleware/auth.ts`
- Verifies Bearer token from Authorization header
- Decodes JWT, fetches user from DB, attaches `req.user`
- Reusable `requireAuth` middleware

### 3. Role authorization middleware
- **New:** `apps/api/src/middleware/requireRole.ts`
- Factory function: `requireRole('admin', 'manager')` returns middleware
- Checks `req.user.role` against allowed roles
- Returns 403 if unauthorized

### 4. Update user creation route
- **Modify:** `apps/api/src/routes/users/create.ts`
- Check if any users exist in DB; if none, assign `admin` role
- Otherwise assign `user` role

### 5. Include role in JWT and API responses
- **Modify:** `apps/api/src/routes/auth/login.ts` — add `role` to JWT payload
- **Modify:** `apps/api/src/routes/auth/emailToken.ts` — same
- **Modify:** `apps/api/src/routes/auth/confirmEmail.ts` — same
- **Modify:** user GET/list routes — include `role` in select columns
- **Modify:** `apps/api/src/swagger.ts` — add `role` to User schema

### 6. Protect routes with middleware
- **Modify:** `apps/api/src/routes/users/index.ts`
  - `GET /` — `requireAuth` + `requireRole('admin', 'manager')`
  - `GET /:id` — `requireAuth` + `requireRole('admin', 'manager')`
  - `POST /` — no auth (registration)
  - `PUT /:id` — `requireAuth` + owner-or-admin check (admin can edit any, others only self)
  - `DELETE /:id` — `requireAuth` + owner-or-admin check (admin can delete any, others only self)
- **New:** `apps/api/src/middleware/requireSelfOrAdmin.ts`
  - Checks if `req.user.id === req.params.id` or `req.user.role === 'admin'`
  - Returns 403 otherwise
- **Modify:** `apps/api/src/routes/clubs/index.ts`
  - `GET /` and `GET /:id` — `requireAuth` + `requireRole('admin', 'manager')`
  - `POST /` — `requireAuth` + `requireRole('admin')`
  - `PUT /:id` — `requireAuth` + `requireRole('admin', 'manager')`
  - `DELETE /:id` — `requireAuth` + `requireRole('admin')`
- **Modify:** `apps/api/src/routes/teams/index.ts`
  - `GET /` and `GET /:id` — `requireAuth` + `requireRole('admin', 'manager')`
  - `POST /` — `requireAuth` + `requireRole('admin', 'manager')`
  - `PUT /:id` — `requireAuth` + `requireRole('admin', 'manager')`
  - `DELETE /:id` — `requireAuth` + `requireRole('admin')`

### 7. Update types
- **Modify:** `apps/api/src/types/user.ts` — add `role` field
